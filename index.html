Notable markup differences

  1. New parsing rules for browsers. Both <img src="test.jpg" /> and <img src="test.jpg"> accepted.
  2. New tags: <article> ... <video>, <track>, <audio> ... <canvas> ...
  3. New form controls: email, number, search, url, range, date, time, tel, color
  4. New form control attributes: placeholder, pattern, required, list, min, max, step, etc.
  5. New attributes: charset (on meta), async (on script)
  6. Custom attributes: data-*. Set: data-userid="1234" Get: elem.dataset.userid

HTML markup

  1. DOCTYPE is case insensitive
  2. DOCTYPE is easy to remember
  1. meta charset is missing
  2. Even without DOCTYPE or old DOCTYPE, Chrome, Opera, Firefox interpret new HTML5 attributes
  1. 'article' can be nested. Nested articles are like a blog posts with comments.
  2. 'article' is similar with 'entry' or 'item' from XML feeds.
  3. 'article' can have a 'header' and footer' or multiple 'sections'.
  4. 'section' is for thematic grouping of elements.
  5. don't use 'section' for styling, use 'div' and 'span' instead
  6. use 'nav' only for major link navigation.
  7. don't use 'nav' for the links in the 'footer'. 'footer' is sufficient.
